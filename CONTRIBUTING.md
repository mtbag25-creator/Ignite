# Contributing to Ignite Markets

Thank you for helping make Ignite Markets better!

## 📝 How to Contribute

### Report a Bug
1. Check if issue already exists
2. Open a GitHub Issue
3. Include:
   - Device/browser you're using
   - Steps to reproduce
   - What you expected vs. what happened
   - Screenshots if helpful

### Suggest a Feature
1. Open a GitHub Discussion
2. Describe the feature
3. Explain why it would help users
4. Share any design ideas

### Submit Code
1. Fork the repository
2. Create a feature branch: `git checkout -b feature/awesome-feature`
3. Make your changes
4. Test on mobile AND desktop
5. Commit: `git commit -m "Add awesome feature"`
6. Push: `git push origin feature/awesome-feature`
7. Open a Pull Request

## 📋 Code Style

- Use 2-space indentation
- Add comments for complex logic
- Test on mobile devices
- Keep functions under 100 lines
- Use descriptive variable names

## 🧪 Testing Checklist

Before submitting PR, verify:
- [ ] Works on Chrome (desktop)
- [ ] Works on Firefox (desktop)
- [ ] Works on Safari (mobile)
- [ ] Works on Android browser
- [ ] No console errors
- [ ] Responsive layout 320-1440px
- [ ] Email verification works
- [ ] Admin panel functions

## 📧 Development Email Setup

For testing email features:
1. Create a free Gmail account
2. Enable 2FA
3. Get App Password
4. Add to `.env.test`

## 🚀 Release Process

Maintainers follow semver:
- `1.0.0` = major (breaking changes)
- `1.1.0` = minor (new features)
- `1.0.1` = patch (bug fixes)
